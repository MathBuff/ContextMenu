<br> behavior in different containers

Normal block div — <br> creates a real line break
one two
three four ← line break between "two" and "three"
Flex container — <br> becomes inline/anonymous inside flex; typically does not force a new flex line
one two
three four ← you may see no forced new row here
If you enable wrapping the flex items will wrap based on width and not because of <br>.
one two
three four ← flex-wrap: wrap controls wrapping here
Tips
  • Use <br> for semantic inline line breaks inside normal text content.
  • Avoid using <br> to control layout inside display:flex; use flex-wrap, widths, or block elements instead.
  • To force a new row in flex reliably add a block-level element sized 100% width or toggle flex-wrap on the container.
